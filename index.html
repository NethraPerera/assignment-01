Let's learn about INTERNET

  1. What is INTERNET?

The Internet is a global system of connected computer networks that lets devices communicate and share information.

  • It’s a network of networks: many smaller networks (home, school, company, mobile) are linked together to form one very large network that reaches the whole world.
  • Devices on the Internet (computers, phones, servers, IoT devices) exchange data in small pieces called packets, using agreed rules (protocols) so both sides understand each other.

  1. How it started and why it matters

The internet started in the late 1960s as a U.S. Department of Defense project named ARPANET.

  • Created to build a decentralized, fault-tolerant communication network, it allowed university and military computers to share data. The system officially became the modern internet on January 1, 1983, with the adoption of TCP/IP communication standards

  1. How the Internet works (Client-Server model, HTTP, DNS).

The Internet works through a client-server model

  • your browser or app is the client, and a computer that stores the website or data is the server. The client sends a request, the server processes it,and then sends back a response.
  • The client sends a request using HTTP. Before the request is sent, DNS translates the website name into an IP address so the browser can find the correct server. The server then sends back the requested information to the client.

Client-Server model

  • In simple words, the client asks and the server answers. For example, when you open a website, your browser sends a request to the website’s server, and the server sends back the page content such as HTML, CSS, JavaScript, images, or data.

HTTP

  • HTTP stands for Hypertext Transfer Protocol. It is the set of rules used for communication  
between the client and the server on the web, so both sides know how to send and understand 
   requests and responses.A simple HTTP flow looks like this:
    1. You type a website address in the browser.
    2. The browser sends an HTTP request.        
    3. The server receives the request.
    4. The server sends an HTTP response.
    5. The browser shows the webpage.

DNS

  • DNS stands for Domain Name System. It works like the phone book of the Internet by changing a website name like google.com into an IP address, which is the real numerical address of the server. This is important because computers communicate using IP addresses, but people find names easier to remember. So before your browser can contact the server, it first asks DNS for the server’s IP address.
  • sSimple example When you enter www.example.com in your browser: 
        1) DNS finds the IP address of the website.
    2) The browser connects to that server.
    3) HTTP is used to request the page and receive the response.
